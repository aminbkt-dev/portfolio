# Brief de création du site portfolio freelance — Amin Bakhti

## Objectif du site

Créer un site portfolio **one-page**, moderne, lisible, premium et orienté business pour présenter le profil freelance de **Amin Bakhti**, ses compétences en data, son projet principal **AmeOS**, et donner envie à des clients potentiels de le contacter.

Le site ne doit **pas** ressembler à un portfolio étudiant ni à un simple CV web.  
Il doit donner une impression de **crédibilité, clarté, autonomie et professionnalisme**.

Le but principal est de :
- présenter clairement le profil
- montrer un vrai projet concret (AmeOS)
- faire comprendre la valeur apportée
- servir de support pour la prospection freelance
- inspirer confiance en moins de 30 secondes

---

## Positionnement à refléter

Amin est un **Freelance Data Analyst / Analytics Engineer**.

Il se positionne comme un profil hybride capable de :
- structurer les données
- concevoir des pipelines ETL
- automatiser des flux de données
- modéliser une base analytique
- créer des dashboards Power BI
- produire des analyses business exploitables

Le site doit bien refléter ce positionnement hybride :
- **analyse**
- **pipeline**
- **automatisation**
- **pilotage de la performance**

---

## Ton du site

Le ton doit être :
- professionnel
- moderne
- crédible
- business-oriented
- fluide
- accessible
- pas trop corporate
- pas trop marketing

Éviter :
- les formulations trop agressives
- le ton “guru”
- le ton “startup bullshit”
- les phrases trop longues
- le vocabulaire trop abstrait
- les expressions creuses

Le français doit être naturel, propre, clair, sans mélange inutile français/anglais.  
Les mots techniques comme SQL, ETL, Power BI, API peuvent rester en anglais.

---

## Style visuel attendu

Le site doit être :
- one-page
- responsive
- moderne
- minimal
- aéré
- premium
- très lisible
- agréable à parcourir
- sans surcharge visuelle

Direction visuelle :
- fond clair / blanc
- cartes arrondies
- ombres légères
- typographie nette
- hiérarchie visuelle forte
- navigation sticky en haut
- scroll fluide
- sections bien espacées
- animations très légères uniquement
- aucun effet flashy
- aucun style trop “template étudiant”

Éviter :
- trop de couleurs
- dark mode
- trop d’animations
- sections inutiles
- testimonials fake
- logos clients fictifs
- pricing
- blog

---

## Stack technique à utiliser

Créer le site avec :
- React
- Tailwind CSS
- Vite

Si possible :
- un composant principal unique ou une structure simple
- code propre et lisible
- aucune dépendance inutile
- pas de backend
- pas de CMS
- pas de base de données
- pas d’API externe nécessaire

Le site doit être facile à modifier ensuite.

## 4. Projet principal : AmeOS

### C’est la section la plus importante du site

Cette section doit être la plus forte, la plus détaillée et la plus convaincante.

### Ce qui doit être clairement compris
AmeOS n’est pas un projet fictif.
C’est un système data réel conçu par Amin pour piloter **sa propre marque e-commerce, Ame Sportswear**.

Il faut absolument que le visiteur comprenne :
- que c’est sa propre marque
- que le système a été conçu dans un contexte réel
- qu’il répond à de vrais enjeux business
- que ce n’est pas juste un exercice technique

### Titre
Projet principal : AmeOS

### Sous-titre
Système data complet conçu pour piloter la performance d’une activité e-commerce

### Éléments à raconter
Le texte doit expliquer que :
- Amin a conçu et développé AmeOS dans le cadre du développement de sa propre marque e-commerce, Ame Sportswear
- le but était d’avoir une vision fiable et exploitable de la performance (ça repond a un serieux probleme qui etait qu'avant de creer ce systeme, il suivait sa data plus difficilement, tout etait eparpillé et il n'avait pas de vision claire sur des data qu'il jugeait importantes)
- le système permet de suivre les ventes, les coûts, la rentabilité, les retours et la performance commerciale
- plusieurs sources sont centralisées
- les données sont transformées et structurées pour l’analyse
- le système alimente ensuite des dashboards Power BI

### Stack et logique du projet à mentionner
- Shopify API → source de ventes, produits, variants, retours, paiements
- compte bancaire -> fichiers csv → source des dépenses
- n8n → synchronisation / ETL / automatisation
- JavaScript dans les nodes n8n pour préparer et transformer les données
- PostgreSQL / Supabase → base analytique centrale
- Power BI → dashboards analytiques

### Logique métier à mentionner
Le projet doit montrer qu’il ne s’agit pas seulement de connecter des outils, mais de structurer une logique analytique réelle.

Il faut mentionner :
- l’intégration des ventes, dépenses,retours et stocks
- la modélisation de la base
- la logique d’attribution des ventes par collection
- les règles garantissant la fiabilité des indicateurs
- la rentabilité produit / collection
- le pilotage business

### Ce que le projet démontre
Créer une sous-section claire :
**Ce que ce projet démontre**

Avec des points du type :
- Conception d’un pipeline ETL de bout en bout
- Modélisation d’une base analytique
- Structuration de règles métier
- Développement de dashboards Power BI
- Automatisation des flux de données
- Pilotage de la rentabilité et de la performance

### Visualisation d’architecture
Ajouter un visuel simple et propre montrant :
**Shopify API → n8n → PostgreSQL / Supabase → Power BI**

Pas besoin de schéma très complexe.
Le rendu doit être propre, moderne et compréhensible immédiatement.

### Important
Cette section doit donner envie à un prospect de se dire :
**“OK, il a déjà construit quelque chose de sérieux en conditions réelles.”**
